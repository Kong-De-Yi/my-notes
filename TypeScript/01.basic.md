# 简介

## TS是JS的静态类型检查器

- 静态类型检查：在不运行代码的情况下根据被操作的值的类型来检测错误
- 非异常失败：捕获 ECMAScript 规范之外的bug

## TS是JS的类型化超集

- JS语法是合法的TS
- TS添加了如何使用不同类型的值的规则
- TS保留JS的运行时行为
- TS在编译完成后会擦除类型

## 编译器tsc

- 将TS转译为JS
- 报错不影响产出文件
- 擦除类型
- 降级
- 可配置类型检查的严格性 strict:true
  - noImplicitAny:在遇到被隐式推断为any类型的变量会抛出错误
  - strictNullChecks:必须处理null和undefined

# 类型系统

## 类型推断

- TS可以自动推断变量的类型

```typescript
// let hellWorld:string
let helloWorld = "Hello World";
```

## 常见类型

- 基本类型：string,number,boolean
- 数组：type[]
- any：禁用类型检查

### 对象类型

```typescript
function printCoord(pt: { x: number; y: number }) {
  console.log("The coordinate's x value is " + pt.x);
  console.log("The coordinate's y value is " + pt.y);
}
printCoord({ x: 3, y: 7 });
```

## 类型注解

- 变量

```typescript
let myName: string = "Alice";
```

- 函数参数

```typescript
function greet(name: string) {
  console.log("Hello," + name.toUpperCase() + "!!");
}
```

- 函数返回值

```typescript
function getFavoriteNumber(): number {
  return 26;
}
```

- 匿名函数

```typescript
const names = ["Alice", "Bob", "Eve"];
names.forEach(function (s) {
  console.log(s.toUpperCase());
});
names.forEach((s) => {
  console.log(s.toUpperCase());
});
```

## 定义类型

- 使用接口(interface)和类型(type)

```typescript
interface User {
  name: string;
  id: number;
}

const user: User = { name: "Hayes", id: 0 };

class UserAccount {
  name: string;
  id: number;
  constructor(name: string, id: number) {
    this.name = name;
    this.id = id;
  }
}

const user: User = new UserAccount("Murphy", 1);
```

## 组合类型

### 联合

- 声明类型可以是多种类型中的一种

```typescript
// 组合基本类型
type MyBool = true | false;

// 描述字面量的合法值
type WindowStates = "open" | "closed" | "minimized";
type LockStates = "locked" | "unlocked";
type PositiveOddNumbersUnderTen = 1 | 3 | 5 | 7 | 9;

// 描述函数参数
function wrapInArray(obj: string | string[]) {
  if (typeof obj === "string") {
    return [obj];
  }
  return obj;
}
```

### 泛型

- 为类型提供变量

```typescript
type StringArray = Array<string>;
type NumberArray = Array<number>;
type ObjectWithNameArray = Array<{ name: string }>;
```
